# wiki-lint

위키 전체를 감사(audit)해 문제 목록을 리포트한다. **자동 수정은 사용자 확인 후에만** 한다.

## 점검 항목

1. **모순** — 페이지 간/소스 간 충돌하는 주장. (기존 `⚠️ 모순` 플래그 포함)
2. **오래되거나 불확실한 주장** — 근거가 약하거나 최신성이 의심되는 내용.
3. **고아 페이지** — 어떤 페이지에서도 링크되지 않고 index.md에도 없는 페이지.
4. **누락된 크로스레퍼런스** — 서로 언급하는데 링크가 없는 개념 쌍.
5. **index.md 불일치** — 실제 `wiki/` 파일과 `index.md` 목록의 차이(누락/유령 항목).

## 절차

1. `wiki/` 전체와 `index.md`를 훑어 위 5개 항목을 점검한다.
2. 항목별로 **파일 경로와 함께** 발견 사항을 리포트로 출력한다. 문제 없으면 "이상 없음"으로 표기.
3. 수정 제안을 제시하고, 사용자가 승인하면 반영한다 (index 동기화, 링크 추가 등).
4. `wiki/log.md`에 `## [YYYY-MM-DD] lint` 로 감사 결과 요약을 append한다.
