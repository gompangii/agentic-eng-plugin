# wiki-query

위키를 근거로 질문에 답한다. 추측이 아니라 **위키에 적힌 내용 + 인용**으로 답하는 것이 원칙이다.

인자: `<질문>`

## 절차

1. **검색**
   - `index.md`로 관련 카테고리/페이지를 좁힌 뒤, 관련 `wiki/` 페이지들을 읽는다.
   - 필요하면 개념 페이지의 크로스레퍼런스를 따라간다.

2. **종합 답변**
   - 찾은 내용을 종합해 한국어로 답한다.
   - **각 주장마다 근거 페이지를 인용 링크로 표기**한다. 예: `... 이다 ([하네스](wiki/concepts/harness.md)).`

3. **공백 표시**
   - 위키에 근거가 없는 부분은 "위키에 아직 없음"이라고 명시한다. 지어내지 않는다.
   - 필요하면 어떤 raw를 ingest하면 채워지는지 제안한다.

4. **답변 저장 (선택)**
   - 재사용 가치가 있는 답변이면 `wiki/qa/`에 페이지로 저장할지 사용자에게 제안한다.
   - 저장 시: frontmatter(category: qa) 작성 → `index.md`의 Q&A 섹션에 추가 →
     `wiki/log.md`에 `## [YYYY-MM-DD] query | <질문 요약>` append.
