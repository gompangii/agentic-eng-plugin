# wiki-ingest

새 원본 소스 하나를 위키에 흡수(ingest)한다. **소스 1개당 관련 페이지 여러 개(10±)를 touch**하는 것이 목표다.

인자: `<raw 파일경로>` (예: `raw/ep01/script.md`). 경로가 주어지지 않으면 아직 ingest되지 않은 `raw/` 파일을 확인하고 사용자에게 물어본다.

## 절차

1. **소스 읽기 (수정 금지)**
   - 주어진 `raw/` 파일을 읽는다. raw는 불변 — 절대 편집하지 않는다.

2. **추출**
   - 핵심 개념·용어, 원리/주장, 메타포, 구체 사례를 뽑아낸다.
   - 각 개념이 concept / topic / example 중 어디에 속하는지 분류한다.

3. **기존 위키 확인**
   - `index.md`와 `wiki/` 하위 파일을 훑어 이미 있는 개념인지 확인한다.
   - 있으면 **갱신(merge)**, 없으면 **새 페이지 생성**. 개념당 1파일 원칙 유지.

4. **페이지 작성** (CLAUDE.md의 페이지 규약·frontmatter 준수)
   - `wiki/concepts/`: 개념 페이지. 정의 → 원리 → 예제 → 관련 개념 링크 순.
   - `wiki/topics/`: 이 소스(에피소드)의 종합 개요. 다룬 핵심 개념 페이지들로 링크.
   - `wiki/examples/`: 사례·실습·메타포.
   - frontmatter의 `sources`에 이 raw 경로를 추가, `updated`는 오늘 날짜.

5. **크로스레퍼런스 & 모순**
   - 관련 개념끼리 상대경로 마크다운 링크로 연결한다.
   - 기존 페이지와 충돌하는 내용이면 `> ⚠️ 모순: ...` 블록으로 플래그(양쪽 소스 명시).

6. **index.md 갱신**
   - 새로 만들거나 바뀐 페이지를 해당 카테고리 섹션에 링크 + 한 줄 요약으로 추가/수정.

7. **wiki/log.md 갱신 (append)**
   - `## [YYYY-MM-DD] ingest | <소스 제목>` 헤더 아래에 생성/갱신한 페이지 목록을 적는다.

8. **요약 보고**
   - 생성/갱신한 페이지 수와 목록, 발견한 모순을 사용자에게 간단히 보고한다.
